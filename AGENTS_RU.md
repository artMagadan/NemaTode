# AGENTS_RU.md

Рекомендации для агентов, работающих с этим репозиторием.

## Обзор проекта

NemaTode - легковесный C++11-парсер NMEA и GPS-фреймворк. Публичный API
находится в `include/nmeaparse/`, реализации - в `src/`, а
`demo_simple.cpp` и `demo_advanced.cpp` служат примерами использования.

Этот репозиторий подключен как vendored-зависимость внутри `thirdParty`,
поэтому изменения должны быть точечными. Не делайте крупные переписывания,
если пользователь явно не попросил об этом.

## Сборка и проверка

- Предпочтительная локальная сборка:
  - `cmake -S . -B build_lib`
  - `cmake --build build_lib`
- В `build/` и `build_lib/` уже могут находиться сгенерированные результаты
  сборки. Не редактируйте сгенерированные CMake-файлы вручную.
- В этом checkout нет отдельного набора тестов. Для изменений в парсере
  соберите библиотеку и, если полезно, временно включите или скомпилируйте
  демо-программы, чтобы проверить поведение на включенных файлах
  `nmea_log_*.txt`.

## Соглашения по коду

- Сохраняйте совместимость с C++11.
- Держите публичные заголовки в `include/nmeaparse/`, а соответствующие
  реализации - в `src/`.
- Следуйте существующему стилю: табы часто используются для выравнивания,
  фигурные скобки оформлены компактно, код использует пространство имен
  `nmea`.
- Не добавляйте внешние зависимости без необходимости; сейчас библиотека от
  них не зависит.
- Сохраняйте обратную совместимость API, если задача явно не требует
  ломающего изменения.

## Примечания по CMake

- `CMakeLists.txt` определяет статическую цель `NemaTode` и устанавливает
  заголовки вместе с CMake package config.
- Если добавляются или удаляются исходные файлы или заголовки, обновите списки
  `headers` или `sources` в `CMakeLists.txt`.
- Цели для демо сейчас закомментированы. Не включайте их на постоянной основе,
  если об этом не попросили.

## Гигиена репозитория

- Не коммитьте и не изменяйте сгенерированные артефакты сборки, если
  пользователь не попросил обновить vendored-артефакты.
- Не откатывайте существующие локальные изменения. Этот checkout может
  содержать пользовательские или vendor-изменения относительно upstream.
- Не трогайте `.DS_Store`, сгенерированные файлы CMake cache, собранные
  библиотеки или вывод Xcode project, если задача явно не касается этих файлов.
